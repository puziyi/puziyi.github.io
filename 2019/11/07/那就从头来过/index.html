昨天很意外地，升级系统时它崩掉了，重装遇系列大坑，深陷其中难以自拔……之前写过的文章暂时也没有留存原稿（这点后面详细说），从系统到博客站点，基本全重新搭了一遍，虽然的确丢失了一些东西，踩坑也是真的残存一丝丝的坚强，不过今天终于搞好了，想一想，无非也就是从头再来过而已。

但是，失误及遇到的问题，需要被整理和重视，姑且让我以一篇踩坑日志，作为博客站经受无端“血洗”之后重生的见证与庆祝。

没有谁是一开始就可以把装系统中的各种问题的解答方案揣在兜里，无非是两天装了好几次系统和环境，最后一次意外地顺利点亮了，高兴得记住了惨痛的教训。

同样的事情，每次去做都有新发现和解决老问题，所以打算按照遇到的问题/问题种类/主题相关逐个描述问题情况的逻辑来实现这篇文章（当然有的是这次遇到的，有的可能只是刚好想到就顺手写一下）。

建议单曲循环孙燕姿《遇见》感受刺激同步心跳。

我遇见坑/会有怎样的对白
我等的done/它在多远的未来
我遇见你是/最美丽的意外
总有一天/我的谜底会解开

数据丢失与备份恢复

看到ubuntu 19.10可以更新的系统通知，非常开心地点了升级，在此之前只把昨天改好的代码更新到仓库，出于对自己吸bug体质的忽视，其他的备份操作并没有做，没想到升级失败，大意了。崩了是崩了，但是还得赶紧抢救，于是开始重装，比较好的是，老司机才烧录了一个19.10的启动盘，且通过启动机器的样子推断(参考前几天才研究到的操作系统及硬件的知识)，还能正常进BIOS，于是赶紧借过来开始重装系统。

直到今天早上我都还觉得说，我改好的代码已经提交了的，其他的没有太大问题。但是首先是，我之前下在本地的实验数据也没了，且之前我们服务器数据重置了一次，我得去找远在美国的聪哥拿数据（实属万幸），这个失误有点大；其次是，我本以为一些稿子丢了我可以重写的，而且之前发过的博文原稿我存了，但是刚我才发现，我并没有把之前的博客站代码和原稿存下来，只有Hexo生成的静态文件……所以之前发过的博客现在我手上只有.html文件，内容倒是没丢（还好还好），但今天重新建站也花了有一小会儿，之后再找时间还原原稿吧……

目前想到的应对数据丢失的解决的方法是：一定一定，做到代码、文档、笔记、资料每日更新备份，重要文件起码要保留副本到非系统磁盘上。文件存坚果云，源代码更新后及时同步github/gitlab。

Git: 我不配有姓名的吗？

正常流程：在终端键入Git 安装命令 sudo apt-get install git，输入用户密码，坐等几秒，即可。

出现问题：Package 'git' has no installation candidate

解决方法：

  • 我打开了 Ubuntu Software，等它初始化完成，加载好meta data应用列表这些之后，再执行安装命令就可以了……典型的心急没吃上热豆腐案例
  • 直接换了Ubuntu的软件源，切换成清华Tuna镜像站，更新软件源之后也可以了

Anaconda 系列

安装时候的最后一个决策

Anaconda安装时候最后会问说“要不要帮你写到环境变量里面去呀”，默认是不写，但是实际操作中一般是要写进bashrc里面去，个人觉得小坑

conda deactivate

但是安装完之后,日常使用的terminal里面出现了(base)的前缀，看着真的难受，于是又去bashrc里面补一句conda deactivate，重启终端，就可以了。

网络proxy配置

由于我用来科学学术的小工具不能直接改系统设置，需要在浏览器上手动配置proxy规则，在firefox上直接修改浏览器网络设置，很简单就改好了，但是chrome直接绑定的系统设置，没起作用，就下了个SwitchyOmega管理，这样就可以回学校用图书馆电子文献了。

强行插件安装

打不开chrome插件商店，但在github下好了.crx文件，这种情况下，插件安装操作：chrome://extensions/打开Developer mode然后把下好的.crx解压之后丢进去Load unpacked里面即可。

奇异偏好设置

焦点跟随鼠标

之前读了一本书，书里作者介绍了一个窗口得到焦点的模式，即，鼠标hover的窗口即得到焦点，不需要去拿鼠标点点点就可以唤醒想要使用的窗口，试过之后重度依赖，于是新装系统也要赶紧配置上。

新版的Ubuntu并没有继续用Unity桌面，而是回到了GNOME，在GNOME下管理窗口与鼠标焦点的程序是Tweaks，但是我也没有预装，所以先安装，这个装好之后有一个非常便于操作的GUI，点开在“Windows”选项中把Focus设置为Sloppy即可。

安装命令：sudo apt install gnome-tweak-tool
参考资料：How do I set focus follows mouse in Ubuntu 17.10 and later with GNOME 3?

鼠标滚轮上下控制

之前在实验室的时候，为了跟苹果一体机协同，把鼠标滚轮的上下控制改成“下滚上翻”的模式，然后现在，它回到了大多数机器共同的样子“下滚下翻”，我的确还是不太习惯，虽说目前不是很想改，但也顺手记一下。
