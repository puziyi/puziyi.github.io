入手原因 / 考虑

我之前的上古 kindle 4 被我搞坏而退出历史舞台了，虽然有骚尼板子，但毕竟一方面大不方便揣兜里，另一方面，虽然有人已经提供了刷机方法，但是有点贵不舍得刷，还是老老实实拿着看paper比较好吧。

小的这种电纸书，最主流的考虑肯定是kindle，但是我之前有过了，想换一下，当年刷机也就是多看for kindle，而且我对小米生态链的评价的确会比小米手机的评价高一些，而且安卓系统肯定可以延展的，会比较好玩，于是入了小米多看电纸书（结果还送了我一个大台灯）。

改造结果

既有基础的书架管理、无线传输和百度云盘同步这样的多看原生系统功能，又通过安卓系统安装了个人比较习惯用的网易蜗牛读书水墨版、kindle、扇贝阅读和藏书馆APP，使得我可以在电纸书上继续使用对应的账户进度和权益，而且我还加装了文件管理器，使得电纸书支持OTC-U盘传输文件，我手机平板上可以看的书基本都可以在这边继续，不同终端设备之间冗余高一点，切换也流畅一些。

其实一开始没太多想，照着知乎上说的，连电脑版的360或者豌豆荚，然后把桌面和要用的软件给装上就行

一个插曲

我还特意去网吧弄了一下，结果回来一看，文件管理器和阅读软件都读不到设备的存储器，自带的蓝夜和无线传书都有问题，内部存储器的状态显示的是未挂载，其实当时是想退货了的，但是我毕竟刷了机，要退的话，恢复出厂设置咯，但是就给恢复好了……我又懒得去网吧用windows电脑，想了下ubuntu肯定也可以引导安装apk的，所以go on

Ubuntu 工具及命令

归根到底，是安卓系统开发调试的问题。Linux肯定也可以启动某种服务，以使得可以在电脑上引导apk安装进安卓设备中的。

  • [安卓设备] 进入开发者模式，启动USB调试，通过USB线插到电脑上，USB传输功能选择”传文件“；
  • [电脑终端] 安装adb，sudo apt install adb；
  • [电脑终端] 检测设备连接，adb devices。如果成功会有：List of devices attached和 xxxxxxxxxx device这样的两行显示设备连接情况；
  • [电脑终端] 引导APK安装到设备中，adb install-multiple -r -t [pathtoapk]
  • [电脑终端] 引导设备重启，adb reboot
  • [安卓设备] 断开电脑连接，拔掉线

引导过程详细参考 >> Ubuntu 下 通过 ADB 安装 Apk 和导出手机中的 Apk

应用安装顺序

桌面软件 >> E-Ink Launcher

悬浮球下载 >> 悬浮球.apk， 这里感谢一下悬浮球的开发者时鹏亮，很好用，个人觉得比zhihuer提到的虚拟按键大师・微桌面简洁方便很多

方案一： 先安装桌面，再安装豌豆荚之类的手机助手，重启设备，通过豌豆荚下载需要的阅读软件，其中需要给豌豆荚授予安装第三方应用的权限，没有返回键，但是可以直接回桌面，再进入豌豆荚继续安装；之后就可以开始使用了

方案二： 先安装桌面，再直接安装其他的阅读软件和悬浮球，重启设备，开始使用

方案三： 悬浮球first的安装流程。这个方案是我在挑选悬浮球的时候，发现有前辈其实也写了基于终端命令行的经验，方案流程其实类似方案一，只是他是基于Windows系统，然后悬浮球作为安装入口这样而已。

支持 OTG

自带的文件管理器我尝试了一下，对U盘的识别还是有点问题，于是额外装了一个ES文件浏览器，加上之后插U盘上去就会提示是否用ES文件浏览器打开U盘，打开之后可以正常地在U盘和电纸书之间移动文件，可以直接选择打开方式，也可以把书放在Books里面，回到多看桌面里面去看，但是要刷新一下才能看得到新加入的书籍。
