Cool, we are about to build sophisticated research tools with TensorFlow(Golang)! However, it is a little bit unfamiliar to me now since I didn’t write big deep learning demos for nearly a year, and even I never use TensorFlow in Golang. So TensorFlow once again. “Go” for it :)

Install

At the very beginning, we need a product environment equipped with Golang, TensorFlow-C (oh, yeah, C/CPP environment is must, since TensorFlow has been written in CPP), and TensorFlow-Go package and its dependencies.

Engineers in Google are excellent writers in both code and development documents, so just follow official instructions to download, install and check our dev preparations in order: Golang, TensorFlow-C, TensorFlow-Go

whereis tensorflow/go/core

But I met awkward warning info said that I did not have some package in the path github.com/tensorflow/tensorflow/tensorflow/go/core when the public repository of TensorFlow in Github doesn’t have this particular dir either. So, where is it?

I found a guy shared his similar situation in his blog, and the main problem is nested repository calls, and he dealt with it as the following steps:

  1. Run go get github.com/tensorflow/tensorflow/tensorflow/go
  2. cd $GOPATH/src/github.com/tensorflow/tensorflow/tensorflow/go
  3. git checkout r1.11
  4. Run go get github.com/tensorflow/tensorflow/tensorflow/go AGAIN

Then I did the same operations and the “hello_tf.go” demo ran successfully, and the message Hello from TensorFlow version 1.15.0 showed up in my terminal.
